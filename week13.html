Best Practises & Common Mistakes

  • I've learnt about the common mistakes made by others for FED assginment 1 and how to correct them.

  • For instance, our commit messages must be meaningful and we should commit our work early and often.

  • File structure must be neat and have good naming convention.

  • Deployment to GitHub Pages is a must, wireframes are not the final design - but an initial concept/structure.

  • Visuals should be pleasant, code quality must be there, README file should have appropriate structure and writing.
